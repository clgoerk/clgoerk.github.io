getFIT – iOS Workout Tracking App (SwiftUI)

Capstone Mobile App Project by Chris Goerk

getFIT is an iOS workout tracking app built using SwiftUI as my capstone project for the Mobile & Web Development program. The app dynamically fetches 1400+ exercises from a remote API, allowing users to plan workouts, log exercises, track progress, and visualize performance over time using charts in a clean, intuitive interface.

Project Type: Capstone Project (Software & Web Development)
Role: Sole Developer (Design & Development)
Platform: iOS (SwiftUI)
Status: Completed

The Problem

Many workout tracking solutions are either too complex, cluttered, or make logging workouts slow and frustrating. I wanted to build a focused, clean app that makes workout logging fast while still providing meaningful progress tracking.

The Goal

Build a clean, mobile-first workout tracking app that allows users to quickly create routines, log workouts, and view progress over time with clear visual feedback.

The Solution

I designed getFIT using SwiftUI with a simple, structured flow: select exercises, log sets and reps, and review progress through history and charts. The app focuses on speed, clarity, and minimizing friction during workouts.

Data & API Integration

Instead of using hardcoded or local data, getFIT consumes a remote exercise API containing over 1400 exercises. The app fetches, parses, and displays this data dynamically, allowing users to browse a large exercise library, view details, and build workouts from real-world data.

This required implementing asynchronous network requests, JSON decoding, error handling, and state-driven UI updates in SwiftUI.

About This SwiftUI Fitness App

This project demonstrates my ability to design and build a complete iOS application using SwiftUI, including multi-screen navigation, state management, data visualization, and reusable UI components. It was built as part of my software development portfolio to showcase real-world mobile app development skills.
